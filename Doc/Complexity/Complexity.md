# 時間複雜度與空間複雜度

本文件整理演算法分析常用的時間複雜度與空間複雜度概念。

## 時間複雜度

時間複雜度描述輸入規模 `n` 增加時，演算法執行時間的成長趨勢，常用 Big O 表示上界。

### 常見時間複雜度

- `O(1)`：常數時間，輸入變大時步驟數幾乎不變
- `O(log n)`：對數時間，每次將問題規模縮小（例如二分搜尋）
- `O(n)`：線性時間，步驟數與輸入大小成正比
- `O(n log n)`：線性對數時間，常見於高效率排序
- `O(n^2)`：平方時間，常見於雙層迴圈
- `O(2^n)`：指數時間，常見於暴力遞迴

### 時間複雜度範例

- `O(1)`：陣列以索引取值
- `O(log n)`：已排序陣列的二分搜尋
- `O(n)`：線性搜尋
- `O(n log n)`：合併排序（平均）、快速排序（平均）
- `O(n^2)`：泡沫排序、選擇排序（常見情況）
- `O(2^n)`：列舉所有子集合

## 空間複雜度

空間複雜度描述演算法在輸入規模 `n` 下需要的額外記憶體量。

### 常見空間複雜度

- `O(1)`：僅使用固定額外空間
- `O(n)`：額外空間與輸入規模成正比
- `O(n^2)`：需要矩陣等二維結構

### 空間複雜度範例

- `O(1)`：只使用少量變數的迴圈
- `O(n)`：建立長度為 `n` 的輔助陣列
- `O(n^2)`：建立 `n x n` 二維陣列

## 實務提醒

- 複雜度是成長趨勢分析，不是精確執行時間。
- 資料量小時，常數因子可能比 Big O 更關鍵。
- 選擇演算法時，要同時評估時間與空間的取捨。
